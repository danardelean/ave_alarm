EXACT PROTOCOL SEQUENCE: ARM OPERATION
================================================================================

1. WebSocket Connect
   ↓ (No data sent)
2. Receive Version String (raw text, not XML)
3. Validate Version Matches Exactly
4. Send PAIRING Request (type=PAIRING)
5. Receive FILE Event (configuration, base64)
6. Send LOGIN Request (type=MENU, type=TERM_CODE, filter=USER|POWERUSER)
7. Receive LOGIN Response (res=ACCEPTED)
8. [Optional] Receive ANOMS Event (if anomalies exist)
9. [Optional] Send ANOM_ACK (if warning=0)
10. Send DEVICE_CMD (Command=C00101, AREAS=mask)
11. Receive STATE Event (updated area states)
12. Display EXIT DELAY countdown (from texit attribute)
13. Receive STATE Event (final armed state)

CRITICAL BYTE SEQUENCE:
0x02 + [XML with STX/ETX inside] + 0x03

================================================================================
FRAME FORMAT SPECIFICATION
================================================================================

REQUEST FRAME:
0x02 <?xml version="1.0" encoding="utf-8"?><Request ...>...</Request> 0x03

RESPONSE FRAME:
0x02 <?xml version="1.0" encoding="utf-8"?><Response ...>...</Response> 0x03

EVENT FRAME:
0x02 <?xml version="1.0" encoding="utf-8"?><Event ...>...</Event> 0x03

VERSION HANDSHAKE (FIRST MESSAGE FROM PANEL):
[Raw text version string, NOT XML, NOT framed]
Example: "AF927v1.0.0" or similar

================================================================================
AREA MASK ENCODING
================================================================================

Format: 6-character string where:
- Position 1-6 = Areas 1-6
- Value = Area number (1-6) if included, '-' if excluded

Examples:
"123456" → All areas selected
"123---" → Areas 1, 2, 3 selected
"---456" → Areas 4, 5, 6 selected
"1-3-5-" → Areas 1, 3, 5 selected
"------" → No areas selected

IMPORTANT:
- Position 1 can only be '1' or '-' (not any other digit)
- Position 2 can only be '2' or '-' (not any other digit)
- Position 3 can only be '3' or '-' (not any other digit)
- And so on for positions 4, 5, 6
- Total length MUST always be 6 characters

================================================================================
LOGIN FLOW DETAILS
================================================================================

TERM_CODE (for arm/disarm):
- Filter: USER|POWERUSER
- Type: TERM_CODE
- Purpose: User PIN authentication for operations
- Response includes: area_ins, area_dis masks

SETTINGS (for panel configuration):
- Filter: POWERUSER|INST (varies)
- Type: SETTINGS
- Purpose: Admin PIN for settings access
- Response includes: different permission set

SETTINGS_INST (for installer):
- Filter: INST
- Type: SETTINGS_INST
- Purpose: Installer-level configuration
- Response includes: full system access

================================================================================
STATE EVENT MEANINGS
================================================================================

GLOBAL STATE (Device="AL002"):
- S00120 = Armed/ON (st=2 on all areas)
- S00121 = Disarmed/OFF (st=0 on all areas)
- S00122 = Arming/PART (st=1, has texit countdown)
- S00127 = Alarm triggered

AREA STATE (Area id="1-6"):
- st="0" = Disarmed
- st="1" = Arming (exit delay in texit attribute)
- st="2" = Armed

EXIT DELAY (texit attribute):
- Present only when st="1"
- Value in seconds (typically 88 seconds)
- Countdown display: decrement every second until 0
- When texit reaches 0: area fully armed (transitions to st="2")
